N ARCHIVE ─ OPERATION MANUAL

GitHub公開マニュアル

作ったHTMLを世界に置いて、アプリのように使うまで（2026.07 奈々の実体験版）

全体像 ─ 何をしているのか

HTMLファイル → GitHub（置き場）→ Pages（公開スイッチ）→ URL誕生 → ホーム画面アイコン化

GitHubは「ファイルの置き場」、Pagesは「それをWebページとして公開する機能」。URLは一度作れば変わらず、中身のファイルを上書きするだけで図鑑が育つ。

初回セットアップ（一度だけ）

  1. Safariで github.com → Sign up※アプリ版GitHubは今回の作業に不向き。必ずブラウザで。ユーザー名はURLに入るのでシンプルに。
  2. 右上「＋」→ New repository名前は半角英数（例：mikaku-zukan）。これもURLの一部になる。
  3. 必ず「Public」を選んで CreatePrivateだと無料プランではPagesが使えない（今回ここでつまずいた）。
  4. 「uploading an existing file」→ index.html を選ぶ → Commit changesファイル名は後述の通り厳守。
  5. Settings → 左メニュー PagesSourceは「Deploy from a branch」のまま。Branchを「main」にして Save。
  6. 1〜3分待つ → 「Your site is live at …」が出たら成功出なければページを再読み込みしてもう一度Pagesを開く。
  7. URLを開く → 共有ボタン →「ホーム画面に追加」アイコン登録は一度きり。以後は中身を更新してもやり直し不要。
URLの式：https://ユーザー名.github.io/リポジトリ名/

更新のたびにやること（ループ）

  1. Claudeに図鑑の最新index.html＋新レシピ素材を渡す「既存の構成とデザインを守ってNo.◯◯◯を追加、index.htmlで出力して」と指示。
  2. できた index.html を保存
  3. GitHubのリポジトリ → Add file → Upload files → Commit同じファイル名なら自動で上書きされる。
  4. 1〜2分待って図鑑アイコンから開く → 更新されてる

更新が反映されないときのチェック（2026.07.17追記）

上書きしたのにページが古いまま。原因はほぼこの3つ。上から順に確認する。

  1. リポジトリ側が本当に新しいか見るGitHubのCode画面で index.html を開いて中身を確認。古ければアップロードし直し。「新ファイルを追加」でも古い index.html が残っていると、トップは古い方を表示し続ける。
  2. Actionsタブでビルド完了を確認リポジトリの「Actions」タブを開き、最新の実行に緑のチェック✅が付いていれば反映済み。黄色は進行中（数分待つ）、赤は失敗。
  3. ブラウザのキャッシュを疑う1と2がOKなのに古いままなら、犯人はほぼ端末側のキャッシュ。プライベートブラウズ（シークレットモード）でURLを開くと、キャッシュなしの素の状態で確認できる。それで新しく見えていれば、時間経過か再読み込みで通常表示も直る。
「GitHub側」と「自分の端末側」を分けて考える1・2はGitHub側のチェック、3は端末側のチェック。どちらの問題かを切り分ければ、慌てずに対処できる。

つまずきポイント全記録

ファイル名は「index.html」以外NGカッコ・スペース・日本語が入ると別ファイル扱いになり、ページが真っ白になる。「index（n図鑑HTML）Git Hub.html」で実際に真っ白になった。半角の index.html 厳守。
「Unpublish site」を押さない押すと公開が解除されて真っ白に戻る。押してしまったら：Pages → Branchを一度None → mainに戻す → Save で再公開できる。
Sourceは「Deploy from a branch」「GitHub Actions」に切り替えると難しい設定画面に入ってしまう。触ってしまったらプルダウンで戻すだけでOK。
Saveボタンがグレーなのは正常「変更がない＝押す必要がない」の意味。壊れていない。
真っ白でもすぐ絶望しない反映には1〜3分かかる。リロード→それでもダメならCode画面でindex.htmlの存在とファイル名を確認する。
HTMLをファイルアプリで直接開くと動かないiPadのプレビューはJavaScriptを実行しないため。スワイプ等が動くのはブラウザ（＝URLから開く）だけ。
Publicの意味を忘れないURLを知っていれば誰でも見られる。料理はOK、個人的な軌跡図鑑（N ARCHIVE）を置くときは要相談。
